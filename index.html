iOS解决exc_bad_access

下午在使用NJKWebViewProgress的时候，遇到了exc_bad_access错误。
于是找了下怎么debug exc_bad_access错误的方法，以免遗忘。

下面是报错的源代码和弹出错误的场景：

报错的图片：

就是一个加载网页的webview，NJKWebViewProgress的使用也与官方基本一致。

解决步骤：
1.Edit Schema —> Run —> Diagnostics
选中Enable Zombie Objects
然后再运行项目。
这时重现bug，console会出现错误log：
[NJKWebViewProgress respondsToSelector:]: message sent to deallocated instance 0x15ecb5ad0
通过log得知，因为我向已经被回收的NJKWebViewProgress对象发送消息，从而使程序崩溃。
这句话并没有定位到具体的错误代码，我也没有看出来NJKWebViewProgress怎么就被回收了。所以继续第二步。

2.在xcode项目，使用快捷键command+i,打开instruments，然后再选择Zombies.重复会导致bug的操作，这时在instruments的中间会弹出一个提示框。

点击右侧的箭头，可以看到打开instruments下部分会有NJKWebViewProgress对象引用和释放的纪录。

这里可以看到在错误发生在[UIVebView webView:decodePolicyForNavigationAction]这个语句里。

但真正引起我好奇的是，在viewDidLoad里，NJKWebViewProgress对象怎么就被释放了。双击[WebViewController viewDidLoad]，会进入到xcode里对应的代码。

如图所示，viewDidLoad执行完之后，_progressProxy对象就被释放了。
看了代码发现，NJKWebViewProgress的webViewProxyDelegate和progressDelegate属性都是weak类型的，viewDidLoad方法执行后，_progressProxy自然就被释放了。

3.解决方法，把NJKWebViewProgress对象设为成员即可。

总结：有了instruments，再也不怕遇到exec_bad_access错误了。淡然instruments还有很多作用，比如代码性能的检测，UI绘图的效率等。抽时间一定要掌握instruments！

2016，新工作，新生活，新的开始

终于开始写文章了。
第一次从同事那里得知可以用hexo和Github搭建个人博客是在2015年下半年的事情，那时花了一下午的时间把博客搭了起来，然后想了想，大概没时间写吧，就留了个hello world的blog在那里。等再次想起来写的时候，已经6，7个月过去了。
上一次写类似的blog还是2014年。很差的语文功底，再加上很久没写，每写一句都觉得很困难。但作为今年设定的必须养成的习惯之一，万事开头难，硬着头皮也要写下去了^_^
so，写到习惯，我把2016需要养成的习惯都放到这里。不是为了喊口号，也不是熬鸡汤，只是对自己所定目标的提醒和再次确定。

  1. 写博客，关于技术和生活；
  2. 阅读；

对，就这看上去很宽泛的两条。
其实，从上大学开始，就开始零零散散的写博客。新浪，豆瓣和csdn都留下了我的足迹。但是都没有养成习惯，写不写完全看心情，很容易出现这种断线一年的情况😔。之前写技术博客的经历告诉我，写一篇技术博客，区区几百字，却需要大量的资料查阅和实践，最重要还有对所写内容的理解。写多了之后，又是很好的技术积累，好处多多。一想到这些，变下定养成写博客习惯的决心。
阅读更关键。说也可笑，从小到大，就没有看书的喜好。上学时候，看的最多的可能也就是足球杂志和动漫之类的书籍，专业书就敬而远之。可工作之后，忽然意识到书籍的重要性。便买了很多计算机专业的书，什么代码大全，算法导论，linux内核，什么书好买什么，乱七八糟买了一堆。然后又订了一堆的看书计划。于是四年过去了，一大堆书都堆在那里没看过，然后搬家的时候就书最沉。想到那些看书计划，真是觉得自己可恨又可笑。
都是因为太高估自己，没有定下实际可行的目标。
分享一下我的书单:
凯文凯利的《必然》，《失控》;
iOS相关：《iOS编程》，《iOS开发进阶》，《Effective Objective-C 2.0》,《iOS开发编程》；
android相关:《Java编程思想》，《Head First设计模式》
这么看，又是给自己定下一个不小的目标。。。。但是iOS相关和java设计模式今年一定要搞定。

说一下近况吧。最近趁换工作之际，跟女友好好在外面玩了一个多星期。期间去了重庆，武隆和长沙。重庆之前给我的印象就是火锅，别的并没有什么印象，到了之后才发现超级繁华，甚至感觉比杭州还繁华，街边有很多平易近人的小吃，餐馆，让我感觉比杭州更多了份生活的气息。好吃的东西太多太多，肥肠，老火锅，串串，小面个个都非常对我胃口。顺手58同城了下，房价要比杭州要便宜很多，6，7千一平的样子。对重庆的好感爆棚。
武隆的仙女山，天坑地缝，属于大自然的景观也非常好玩。
长沙感觉就臭豆腐比较好吃。。别的没啥印象，令人失望。

下周一就开始在新单位上班了。希望新的一周工作愉快。
